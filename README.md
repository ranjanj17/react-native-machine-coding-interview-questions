## 📖 What is this repository about?

In recent years, top tech companies have shifted their frontend/mobile interview focus from traditional Data Structures & Algorithms (DSA) to **Machine Coding Rounds**. These rounds are designed to test your actual development skills, architecture knowledge, and ability to build production-ready applications under strict time constraints (usually 45 to 90 minutes).

This repository is your **ultimate playbook** for cracking these rounds. It serves as a curated archive of real-world React Native challenges that developers have faced in actual technical interviews.

## 📦 What does this repository contain?

This repository contains a wide variety of practical problems categorized by their core testing focus. You will find:

### 1. Mini App Clones & Features
- E-commerce cart management flows
- Food delivery tracking screens with maps
- Infinite scrolling social media feeds

### 2. Complex UI & Animations
- Custom bottom sheets and modals
- Gesture-driven swipeable lists (like Gmail or Tinder)
- Shared element transitions and Reanimated challenges

### 3. State Management & Logic
- Building highly optimized search bars with debouncing
- Managing complex global states without prop drilling (Context/Zustand)
- Custom Hooks for fetching, caching, and pagination

### 4. Performance Optimization
- Fixing rendering bottlenecks in large `FlatList` / `FlashList` implementations
- Deeply nested component memoization (`React.memo`, `useMemo`, `useCallback`)

### 5. Offline Storage & APIs
- Handling poor network conditions gracefully
- Offline-first architecture using `AsyncStorage` / SQLite
- Complex form validations and data synchronization

## 🚀 How to use this repository?

To get the most out of this repository, you should clone it and attempt the problems locally.

### Step 1: Clone the Repo
```bash
git clone https://github.com/<your-username>/react-native-machine-coding-interview-questions.git
cd react-native-machine-coding-interview-questions
```

### Step 2: Install Dependencies
```bash
npm install
# or
yarn install
```

### Step 3: Run the Expo App
```bash
npx expo start
```
*Press `a` to open on Android, `i` for iOS simulator, or scan the QR code with the Expo Go app on your physical device.*

### 📂 Folder Structure Navigation
Each interview question is isolated within the `src/` folder (or `app/` if using Expo Router). We recommend reading the problem statement for a specific company/challenge, creating your own branch, and attempting to solve it from scratch before checking the provided optimal solution.

## 🧠 Pro-Tips for Machine Coding Rounds

- **Set a Timer:** Treat it like a real interview. Give yourself 45–60 minutes per challenge.
- **Focus on Architecture:** Don't just dump all your code in `App.js`. Break it down into reusable components, hooks, and constants.
- **Corner Cases Matter:** Always account for loading states, error states, empty states, and network failures.
- **Explain as you code:** If you are practicing for a live-coding round, practice speaking your thoughts out loud.
